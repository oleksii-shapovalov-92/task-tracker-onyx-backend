## Обзор проекта

TaskTracker Backend — это REST API для приложения управления проектами и задачами.

Backend-часть отвечает за регистрацию и авторизацию пользователей, управление проектами, управление задачами, профиль пользователя, подтверждение регистрации по email, сброс пароля, загрузку аватара, валидацию данных и API-документацию.

Это командный учебный проект, разработанный в рамках full-stack обучения.

## Основной функционал

### Авторизация и безопасность

Для защиты доступа к API используется JWT с разделением на два типа токенов.

#### Access Token

- Краткоживущий токен.
- Хранится в HTTP-only cookie.
- Используется для авторизации запросов к защищённым маршрутам.
- Снижает риск компрометации за счёт короткого срока действия.

#### Refresh Token

- Долгоживущий токен.
- Также хранится в HTTP-only cookie.
- Используется для получения нового Access Token без повторного ввода логина и пароля.
- Позволяет поддерживать сеанс пользователя без постоянной повторной аутентификации.

### Механика авторизации

1. Пользователь выполняет вход через `/api/v1/auth/login`.
2. Сервер проверяет email и пароль.
3. После успешного входа сервер выдаёт Access Token и Refresh Token в HTTP-only cookies.
4. Access Token используется для доступа к защищённым маршрутам.
5. После истечения Access Token клиент может запросить новый токен через `/api/v1/auth/refresh-token`.
6. При выходе через `/api/v1/auth/logout` cookies очищаются.

## Возможности проекта

### Аутентификация и регистрация

- Регистрация пользователя.
- Подтверждение регистрации по email-коду.
- Вход пользователя.
- Выход пользователя.
- Обновление Access Token через Refresh Token.
- Сброс пароля через email.
- Ролевая модель доступа.

### Пользователи

- Получение текущего профиля пользователя.
- Обновление профиля пользователя.
- Смена пароля текущего пользователя.
- Загрузка или обновление аватара.
- Получение списка всех пользователей для администратора.

### Проекты

- Создание проекта.
- Получение всех доступных пользователю проектов.
- Получение проекта по ID.
- Обновление проекта.
- Удаление проекта.
- Ограничение доступа к проектам по правилам доступа и владельцу проекта.

### Задачи

- Создание задачи.
- Получение задачи по ID.
- Получение всех задач проекта.
- Обновление задачи.
- Обновление статуса задачи.
- Удаление задачи.
- Ограничение доступа к задачам через правила доступа к проекту.

### Email

- Email для подтверждения регистрации.
- Email для сброса пароля.
- Шаблоны писем на Freemarker.

### Загрузка файлов

- Поддержка загрузки аватара пользователя.
- Конфигурация S3-совместимого хранилища.

### API-документация

- Swagger UI.
- OpenAPI конфигурация.

## Обзор API

### Authentication

| Метод | Endpoint | Описание |
|---|---|---|
| POST | `/api/v1/auth/login` | Вход пользователя |
| POST | `/api/v1/auth/refresh-token` | Обновление Access Token |
| POST | `/api/v1/auth/logout` | Выход пользователя |
| POST | `/api/v1/auth/forgot-password` | Отправка письма для сброса пароля |
| POST | `/api/v1/auth/reset-password` | Сброс пароля |

### Users

| Метод | Endpoint | Описание |
|---|---|---|
| POST | `/api/v1/users/register` | Регистрация пользователя |
| GET | `/api/v1/users/confirm/{code}` | Подтверждение регистрации |
| GET | `/api/v1/users/all` | Получение всех пользователей |
| GET | `/api/v1/users/me` | Получение текущего профиля |
| PATCH | `/api/v1/users/me` | Обновление текущего профиля |
| PATCH | `/api/v1/users/me/password` | Смена пароля |
| PATCH | `/api/v1/users/me/avatar` | Загрузка или обновление аватара |

### Projects

| Метод | Endpoint | Описание |
|---|---|---|
| POST | `/api/v1/projects` | Создание проекта |
| GET | `/api/v1/projects` | Получение всех доступных проектов |
| GET | `/api/v1/projects/{id}` | Получение проекта по ID |
| PATCH | `/api/v1/projects/{id}` | Обновление проекта |
| DELETE | `/api/v1/projects/{id}` | Удаление проекта |

### Tasks

| Метод | Endpoint | Описание |
|---|---|---|
| POST | `/api/v1/tasks` | Создание задачи |
| GET | `/api/v1/tasks/{id}` | Получение задачи по ID |
| GET | `/api/v1/tasks/project/{projectId}` | Получение задач проекта |
| PATCH | `/api/v1/tasks/{id}` | Обновление задачи |
| PATCH | `/api/v1/tasks/{id}/status` | Обновление статуса задачи |
| DELETE | `/api/v1/tasks/{id}` | Удаление задачи |
